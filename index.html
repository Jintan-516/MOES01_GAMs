Introducción

Enfrentamos el problema de explicar una variable de respuesta \(Y\) en relación a un vector de variables explicativas \(\mathbf{X} = (X_1, \ldots, X_k)\).

Evolución de los Modelos

A lo largo de la historia, la modelación estadística ha evolucionado para capturar relaciones más complejas:

  • M. Lineal (1908): \(Y|\mathbf{X} \sim N, \mu = \sum eta_jX_j\)
  • M.L. Generalizado (1972): \(Y|\mathbf{X} \sim EF, g(\mu) = \sum eta_jX_j\)
  • M. Aditivo (1981): \(Y|\mathbf{X} \sim N, \mu = \sum f_j(X_j)\)
  • M.A. Generalizado (1986/90): \(Y|\mathbf{X} \sim EF, g(\mu) = \sum f_j(X_j)\)

Ingredientes de un GAM

Un GAM se compone de tres elementos fundamentales:

  1. Componente aleatoria: \(Y_i|\mathbf{X}_i=\mathbf{x}_i \sim EF( heta_i)\).
  2. Componente sistemática: Relación de la media \(\mu_i\) con un predictor aditivo \(\eta_i\) mediante funciones suaves:
    \[ \eta_i = lpha + \sum_{j=1}^k f_j(x_{ij}) \]
  3. Función liga: La relación ocurre mediante \(g(\mu_i) = \eta_i\).

Construcción del Modelo

Buscamos estimar \(\mathbf{f}\) maximizando la Log-verosimilitud penalizada:

\[ l_P(\mathbf{f}) = l(\mathbf{f}) - rac{1}{2} \sum_{j=1}^k \lambda_j \int [f_j''(x)]^2 dx \]

Donde \(\lambda_j\) controla el compromiso entre ajuste y suavidad (curvatura).

Bases y Splines Cúbicos

Para la implementación computacional se utilizan bases de funciones como los Splines Cúbicos, que permiten representar funciones suaves complejas como combinaciones lineales de bases locales.

Ejemplo Práctico en R

Utilizando el paquete mgcv, el ajuste es directo:

# Código en R
library(mgcv)

# Ajuste con suavizado automático (REML)
modelo_gam <- gam(Viscosidad ~ s(Temperatura, k = 5), 
                   data = datos, 
                   method = "REML")

summary(modelo_gam)
plot(modelo_gam, pages = 1, shades = TRUE)

Este enfoque supera a los modelos lineales transformados en problemas de viscosidad vs temperatura.
